# 1. 비교 목적

창업지원사업 공고문 요약 자동화 챗봇에 가장 적합한 LLM을 선정하기 위해, 동일한 공고 데이터와 동일한 업무 과업 프롬프트를 3개 모델에 입력하고 결과를 비교한다.

비교 목적은 단순히 답변이 자연스러운 모델을 고르는 것이 아니라, 공고명, 접수기간, 신청대상, 제외대상, 제출서류, 교육일정, 문의처처럼 검증 가능한 정보를 정확하게 추출하고, 원문에 없는 정보는 임의로 만들지 않는 모델을 선정하는 것이다.

## 4-1. 평가 축별 채점 근거

- 정확성: 접수기간, 제출서류, 교육일정, 문의처, 제외대상을 원문과 일치하게 추출했는지 확인했다.
- 형식 준수: 요청한 8개 출력 섹션을 모두 유지했는지 확인했다.
- 환각 억제: 원문에 없는 지원금, 교육비, 모집규모를 단정하지 않고 `확인 필요` 또는 보수적 표현으로 처리했는지 확인했다.
- 한국어 자연스러움: 창업 초보자도 이해할 수 있는 문장인지, 과도한 전문용어 없이 설명했는지 확인했다.
- 신청 판단 보조성: 예비창업자 상태에서 필요한 추가 질문을 제시했는지 확인했다.
- 추가 특이사항 처리: 공고번호, 첨부파일, 모집규모 미기재 등 고정 항목 외 정보를 잘 분리했는지 확인했다.

# 5. 모델별 결과 요약

## 5-1. 모델 1: Claude Sonnet 4.6

- 강점: 공고 핵심 정보와 신청대상, 제외대상, 제출서류를 매우 체계적으로 정리했다. 공고번호와 첨부파일 등 추가 확인 정보도 잘 분리했다.
- 약점: 한 줄 요약에서 "교육은 무료 진행"이라고 표현했는데, 제공 데이터에는 교통·숙박비 미지원만 명시되어 있어 교육비 무료 여부는 확인 필요에 가깝다.
- 대표 사례: "모집규모는 확인 필요", "교육비 무상 여부 명시 안 됨"처럼 원문에 없는 정보를 대체로 보수적으로 처리했다.

## 5-2. 모델 2: ChatGPT GPT-5.5

- 강점: 전체 출력 형식을 가장 안정적으로 지켰고, 공고 정보와 사용자 상황을 분리해 신청 가능성 판단을 도왔다. 원문 링크를 근거로 제시해 검증 가능성이 높았다.
- 약점: 출처 링크 표기가 포함되어 보고서에 그대로 붙일 때는 각주 형식을 정리할 필요가 있다.
- 대표 사례: "지원금액: 해당 없음 또는 확인 필요"라고 답해 원문에 없는 금액 정보를 단정하지 않았다.

## 5-3. 모델 3: Gemini 3.5 Flash

- 강점: 요청한 출력 형식을 잘 지켰고, 예비창업자가 이해하기 쉬운 자연스러운 설명을 제공했다. 신청 가능성 판단에 필요한 추가 질문도 적절히 제시했다.
- 약점: 인터넷 검색을 하지 않아 추가 검증 정보가 부족했고, 제공 데이터에 있는 선정절차를 "구체적인 선정 방식 미기재"라고 표현해 정확성에서 감점했다. 또한 "온·오프라인 교육"이라는 표현은 원문 근거가 부족하다.
- 대표 사례: "모집규모 원문 데이터에 기재 없음"처럼 누락 정보는 확인 필요로 잘 표시했다. 다만 2번 공고 핵심 정보 표의 항목 수가 다른 모델보다 적어 정보량 면에서는 아쉬움이 있었다.

# 6. 최종 선정 모델

최종 선정 모델은 **ChatGPT GPT-5.5**이다.

ChatGPT GPT-5.5는 공고명, 접수기간, 신청대상, 제외대상, 제출서류, 교육일정, 문의처를 가장 안정적으로 구조화했다. 요청한 8개 출력 형식을 모두 지켰고, 공고 URL 출처를 함께 제시해 사용자가 결과를 검증하기 쉬웠다.

또한 지원금액, 모집규모처럼 원문에서 명확하지 않은 정보는 단정하지 않고 `해당 없음 또는 확인 필요`로 처리했다. 이는 본 과제에서 요구하는 환각 억제 기준에 가장 잘 맞는다.

Claude Sonnet 4.6도 환각 억제와 형식 준수 측면에서 우수했지만, ChatGPT GPT-5.5가 출처 제시와 핵심 정보 구조화에서 조금 더 안정적이었다. Gemini 3.5 Flash는 형식과 한국어 자연스러움은 좋았지만, 인터넷 검색을 하지 않았고 공고 핵심 정보 표의 정보량이 상대적으로 적어 정확성 검증 측면에서 아쉬움이 있었다. 따라서 창업지원사업 공고문 요약 자동화 챗봇에는 ChatGPT GPT-5.5가 가장 적합하다.
